# 스택에 대한 이해

### 기본 개념

- 한쪽 끝에서만 데이터를 넣고 뺄 수 있는 제한적인 선형 자료구조
- 가장 나중에 들어온 데이터가 가장 먼저 나가는 **LIFO(Last-In, First-Out, 후입선출)** 구조

### 특징 및 장단점

- **장점:** 데이터의 삽입과 삭제가 상단(Top)에서만 일어나므로 속도가 매우 빠름(O(1))
- **단점:** 중간에 위치한 데이터에 직접 접근할 수 없으며, 데이터 탐색을 위해서는 하나씩 꺼내야 하므로 비효율적임

### 주요 활용 사례

- **함수 호출 관리:** 프로그래밍 언어의 런타임에서 함수 콜 스택(Call Stack) 관리 시 사용됨
- **되돌리기(Undo):** 문서 편집기나 브라우저의 뒤로 가기 기능을 구현할 때 활용됨
- **수식 계산:** 후위 표기법(Postfix) 계산 및 괄호 유효성 검사 등에 필수적임
- **재귀 알고리즘:** 재귀 함수의 논리적 흐름을 제어하는 기초가 됨

# 최소 구현 API
### 필수 API
1. `push(item)`
	  - 스택의 최상단에 요소를 추가
2.  `pop()`
	  - 스택의 최상단 요소를 제거하고 반환
	  - 비어 있을 때는 예외/에러 또는 `null`/`None` 등의 정의된 값을 반환하도록 정책을 정의
3. `peek()`
	- 최상단 요소를 제거하지 않고 조회
	- 비어 있을 때의 동작(예외/정의된 값)을 정의
4. `isEmpty()`
   - 스택이 비어 있는지 여부를 반환
1. `size()`
   - 현재 스택에 들어 있는 요소 수를 반환

### 선택 API
1. `clear()`
	- 스택을 비움
2. `capacity()`
  - 고정 크기 배열 기반 구현 시 용량을 출력
  - 혹은 가득 찼는지 여부를 출력

### 구현 시 기본 규칙

- LIFO(Last-In-First-Out) 규칙을 준수해야함
- `pop()`/`peek()`에서의 언더플로우 처리 정책을 일관되게 유지
