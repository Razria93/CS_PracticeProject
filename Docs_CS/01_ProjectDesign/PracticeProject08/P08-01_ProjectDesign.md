# 힙(Heap)에 대한 이해

### 기본 개념

- **Heap**
    
    - 완전 이진 트리(Complete Binary Tree) 기반의 자료구조
    - 부모–자식 간의 **우선순위 조건(heap property)** 을 만족
    - 일반적으로 **배열 기반**으로 구현됨
        
- **Max Heap**
    
    - 부모 노드의 값 ≥ 자식 노드의 값
    - 루트(root)에 항상 **최댓값** 존재
        
- **Min Heap**
    
    - 부모 노드의 값 ≤ 자식 노드의 값
    - 루트(root)에 항상 **최솟값** 존재
        
- **Complete Binary Tree**
    
    - 마지막 레벨을 제외한 모든 레벨이 완전히 채워짐
    - 마지막 레벨은 왼쪽부터 채워짐
    - 배열 인덱스로 표현 가능
        

#### 힙의 배열 표현

배열 인덱스를 0부터 시작한다고 가정하면:

```
parent(i)      = (i - 1) / 2
leftChild(i)   = 2i + 1
rightChild(i)  = 2i + 2
```

트리 구조이지만, 실제 구현은 **배열 기반 인덱스 연산**으로 구성되어 있음

---
### 힙 속성 (Heap Property)

#### Max Heap 예시

```
        50
       /  \
     30    40
    /  \
  10   20
```

- 부모 ≥ 자식
- 루트 = 전체 최댓값

#### Min Heap 예시

```
        10
       /  \
     20    30
    /  \
  40   50
```

- 부모 ≤ 자식
- 루트 = 전체 최솟값

---
### 구조적 특징

- **정렬된 구조가 아님**
    
    - 힙은 “완전 정렬”이 아니라 **부분 정렬 구조**
    - 부모–자식 관계만 보장됨
        
- **우선순위 큐(Priority Queue)** 의 핵심 구현체

---
### 특징 및 장단점

- 장점
    
    - 최댓값 / 최솟값 접근이 O(1)
    - 삽입 / 삭제가 O(log n)
    - 배열 기반 → 메모리 효율적
    - 우선순위 기반 처리에 최적
        
- 단점
    
    - 임의 탐색 O(n)
    - 완전 정렬이 아님
    - 중간 요소 삭제가 비효율적

---
### 시간 복잡도

| 연산              | 시간 복잡도   |
| --------------- | -------- |
| **push (삽입)**   | O(log n) |
| **pop (루트 삭제)** | O(log n) |
| **top (루트 접근)** | O(1)     |

---
### 핵심 연산 원리

#### 1. 삽입 (push)

1. 배열 마지막에 요소 추가
2. 부모와 비교
3. heap property 만족할 때까지 **위로 교환 (Up-Heap / Sift-Up)**

---
#### 2. 삭제 (pop)

1. 루트 제거
2. 마지막 요소를 루트로 이동
3. 자식과 비교
4. heap property 만족할 때까지 **아래로 교환 (Down-Heap / Sift-Down)**

---
### 최소 구현 API

### 필수 API

1. `push(value)`
    
    - 힙에 요소 삽입
    - heap property 유지
        
2. `pop()`
    
    - 루트 요소 제거
    - heap property 재정렬
        
3. `top()`
    
    - 루트 값 반환 (최댓값/최솟값)
        
4. `empty()`
    
    - 비어있는지 여부 반환
        
5. `size()`
    
    - 현재 요소 개수 반환

---

### 선택 API

1. `clear()`
    
    - 전체 초기화
        
2. `buildHeap(array)`
    
    - 기존 배열을 힙으로 변환
        
3. `replace(value)`
    
    - 루트 교체 후 재정렬
        
4. `decreaseKey()` / `increaseKey()`
    
    - 특정 노드 값 변경 후 재정렬

---
### 구현 시 기본 규칙

- 반드시 **Complete Binary Tree** 구조 유지
- 배열 인덱스 연산 정확성 유지
- 삽입 시 Up-Heap
- 삭제 시 Down-Heap
- Max Heap / Min Heap 중 목적에 맞게 선택

---
### 사용 사례

- 우선순위 큐(Priority Queue)
- 스케줄링 알고리즘
- 다익스트라(Dijkstra)
- 힙 정렬(Heap Sort)
- 이벤트 시뮬레이션
- 운영체제 작업 관리
