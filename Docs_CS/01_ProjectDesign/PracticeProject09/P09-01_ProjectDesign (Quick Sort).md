# 퀵소트(Quick Sort)에 대한 이해

### 기본 개념

- **Quick Sort**
    
    - 분할 정복(Divide and Conquer) 기반 정렬 알고리즘
    - 하나의 **피벗(Pivot)** 을 기준으로 배열을 분할
    - 피벗보다 작은 값은 왼쪽, 큰 값은 오른쪽으로 재배치
    - 재귀적으로 부분 배열 정렬
        
- **Pivot**
    
    - 기준 원소
    - 분할 과정의 핵심
    - 선택 방식에 따라 성능에 큰 영향
        
- **Partition**
    
    - 피벗을 기준으로 배열을 두 구간으로 나누는 과정
    - 정렬의 핵심 단계

---
### 퀵소트 기본 흐름

```
1. pivot 선택
2. partition 수행
3. pivot 기준으로 좌/우 재귀 호출
```

---
### 구조적 특징

- **제자리 정렬(In-place Sorting)**
    
    - 추가 메모리 거의 사용하지 않음
    - 공간 복잡도 O(log n) (재귀 스택)
        
- **불안정 정렬(Unstable Sort)**
    
    - 동일 값의 상대적 순서 보장되지 않음

---
### 특징 및 장단점

- 장점
    
    - 평균 시간 복잡도 O(n log n)
    - 메모리 사용량이 적음
    - 실제 구현 시 매우 빠른 성능
        
- 단점
    
    - 최악 시간 복잡도 O(n²)
    - 피벗 선택에 따라 성능 편차 큼
    - 불안정 정렬

---
### 시간 복잡도

|경우|시간 복잡도|
|---|---|
|평균|O(n log n)|
|최악|O(n²)|
|공간 복잡도|O(log n)|

※ 최악의 경우: 이미 정렬된 배열 + 한쪽 끝 피벗 선택

---
### 핵심 연산 원리

### 1. Partition 과정

예시:

```
[8, 3, 5, 2, 7]
pivot = 5
```

→ 분할 후:

```
[3, 2] 5 [8, 7]
```

- 피벗은 최종 위치에 고정됨
- 좌측은 pivot보다 작음
- 우측은 pivot보다 큼
    

---
### 2. 재귀 정렬

```
QuickSort(left)
QuickSort(right)
```

분할이 반복되며 정렬 완성

---
### 최소 구현 API

### 필수 API

1. `quickSort(array, left, right)`
    
    - 분할 정복 정렬 수행
        
2. `partition(array, left, right)`
    
    - 피벗 기준 분할

---
### 선택 API

1. `choosePivot()`
    
    - 피벗 전략 (첫 원소 / 마지막 / 중간 / 랜덤)
        
2. `insertionSort()`
    
    - 작은 구간 최적화
        
3. `threeWayPartition()`
    
    - 중복 많은 경우 최적화

---
### 구현 시 기본 규칙

- pivot 선택 전략 명확히 정의
- partition 정확성 유지
- 작은 구간은 다른 정렬로 최적화 가능
- 재귀 깊이 제한 고려 (tail recursion 제거 가능)

---
### 사용 사례

- 일반 정렬 라이브러리
- 대용량 데이터 정렬
- 비교 기반 정렬의 대표 사례

---
### 정리

| 항목    | Quick Sort | Merge Sort |
| ----- | ---------- | ---------- |
| 평균 시간 | O(n log n) | O(n log n) |
| 최악 시간 | O(n²)      | O(n log n) |
| 공간    | O(log n)   | O(n)       |
| 안정성   | 불안정        | 안정         |
| 실제 속도 | 매우 빠름      | 비교적 안정적    |
