
# 머지소트(Merge Sort)에 대한 이해

### 기본 개념

- **Merge Sort**
    
    - 분할 정복(Divide and Conquer) 기반 정렬 알고리즘
    - 배열을 절반씩 분할
    - 각각 정렬 후 **병합(Merge)** 과정 수행
	
- **Merge**
    
    - 두 개의 정렬된 배열을 하나로 합치는 과정
    - 선형 시간(O(n))에 수행

---
### 머지소트 기본 흐름

```
1. 배열을 반으로 분할
2. 각 부분을 재귀적으로 정렬
3. 두 정렬된 배열을 병합
```

---
### 구조적 특징

- **안정 정렬(Stable Sort)**
    
    - 동일 값의 상대적 순서 유지
        
- **추가 메모리 필요**
    
    - 보조 배열 사용
    - 공간 복잡도 O(n)

---
### 특징 및 장단점

- 장점
    
    - 항상 O(n log n) 보장
    - 안정 정렬
    - 데이터 분포에 영향 받지 않음
        
- 단점
    
    - 추가 메모리 필요
    - 캐시 지역성 낮음
    - in-place 구현 어려움

---
### 시간 복잡도

|경우|시간 복잡도|
|---|---|
|평균|O(n log n)|
|최악|O(n log n)|
|공간 복잡도|O(n)|

---
### 핵심 연산 원리

### 1. 분할 (Divide)

```
[8, 3, 5, 2]
→ [8, 3] / [5, 2]
→ [8] [3] [5] [2]
```

---
### 2. 병합 (Merge)

```
[3] [8]
[2] [5]

→ [3,8]
→ [2,5]
→ [2,3,5,8]
```

- 두 배열을 앞에서부터 비교하며 병합
- 항상 정렬 상태 유지

---
### 최소 구현 API

### 필수 API

1. `mergeSort(array, left, right)`
    
    - 재귀 분할 및 정렬
	
2. `merge(array, left, mid, right)`
    
    - 두 정렬 구간 병합

---
### 선택 API

1. `iterativeMergeSort()`
    
    - 반복 기반 구현
	
2. `inPlaceMerge()`
    
    - 공간 최적화 시도
	
3. `parallelMergeSort()`
    
    - 병렬 처리 확장

---
### 구현 시 기본 규칙

- 분할 기준 정확히 설정
- 병합 시 인덱스 관리 정확성 유지
- 보조 배열 재사용 최적화 가능
- 큰 데이터에 적합

---
### 사용 사례

- 안정 정렬이 필요한 경우
- 외부 정렬(External Sorting)
- 병렬 정렬 구현
- 대규모 데이터 처리

---
### 정리

| 항목    | Quick Sort | Merge Sort |
| ----- | ---------- | ---------- |
| 평균 시간 | O(n log n) | O(n log n) |
| 최악 시간 | O(n²)      | O(n log n) |
| 공간    | O(log n)   | O(n)       |
| 안정성   | 불안정        | 안정         |
| 실제 속도 | 매우 빠름      | 비교적 안정적    |
