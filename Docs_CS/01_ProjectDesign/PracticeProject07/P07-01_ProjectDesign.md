# 큐(Queue)에 대한 이해

### 기본 개념

- **Queue**

	- 선입선출(**FIFO, First-In First-Out**) 구조의 선형 자료구조
	- 먼저 들어온 데이터가 먼저 나감
	- 데이터 접근은 **한쪽은 삽입(rear), 한쪽은 제거(front)** 로 분리됨

- **Front**

	- 가장 먼저 삽입된 요소의 위치
	- `pop()` 또는 `front()`가 참조하는 지점

- **Rear**

	- 가장 마지막에 삽입된 요소의 다음 위치
	- `push()`가 데이터를 추가하는 지점

---
### **FIFO 원칙**

```
push(1)
push(2)
push(3)

pop() → 1
pop() → 2
pop() → 3
```

- **선입선출(First In First Out)의 구조**
- `push` 로 데이터를 추가한 순서대로 `pop`으로 데이터가 제거 됨

---
### 구조적 형태

#### **1. 선형 큐 (Linear Queue)**

- 배열 기반
- rear가 끝에 도달하면 더 이상 삽입 불가
- `pop`이 되어 제거된 `front` 이전의 공간의 재사용이 불가함
- 메모리 공간의 낭비가 발생할 여지가 있음

#### 2. 원형 큐 (Circular Queue)

- 배열을 **환형 구조로 해석**
- `(index + 1) % capacity` 방식으로 순환
- 고정 크기에서 공간 효율성 극대화

#### 3. 연결 리스트 기반 큐

- 동적 할당 기반
- 크기 제한 없음
- front / rear 포인터 유지
- 메모리 단편화 가능성 존재

---
### 특징 및 장단점

- 장점

	- 삽입 / 삭제가 O(1)
	- 구현이 단순
	- 캐시 친화적 (배열 기반)
	- 스케줄링, 버퍼링에 적합

- 단점

	- 임의 접근(Random Access) 불가능 → O(n)
	- 배열 기반은 고정 크기 문제 존재
	- 원형 큐는 상태 관리 실수 시 버그 발생 가능

---
### 시간 복잡도

| 연산    | 시간 복잡도 |
| ----- | ------ |
| push  | O(1)   |
| pop   | O(1)   |
| front | O(1)   |
| empty | O(1)   |

---
### 최소 구현 API

### 필수 API

1. `push(value)`
    - rear 위치에 삽입
        
2. `pop()`
    - front 위치 요소 제거
        
3. `front()`
    - 첫 요소 참조
        
4. `empty()`
    - 비어있는지 여부 반환
        
5. `size()`
    - 현재 요소 개수 반환

---
### 선택 API

1. `back()`
    - 마지막 요소 참조
        
2. `full()` (고정 크기 큐)
    - 가득 찼는지 여부 반환
        
3. `clear()`
    - 전체 초기화
        
4. `resize(newCapacity)`
    - 동적 확장 지원 시 필요
        

---
### 구현 시 기본 규칙

- front / rear 의미를 명확히 정의
- empty / full 조건을 일관되게 유지
- 원형 큐는 반드시 modulo 연산 사용
- capacity - 1 방식인지, size 기반 방식인지 명확히 선택

---
### 사용 사례

- 운영체제 스케줄러
- 네트워크 패킷 버퍼
- 오디오/비디오 스트리밍
- 생산자–소비자 패턴
- 게임 이벤트 큐
